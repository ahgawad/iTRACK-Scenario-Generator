iTRACK - Scenario Generator

Scenario Generator Users Guide

http://localhost:8000/

This link now takes us to this help page.



Web functionalities:

http://localhost:8000/scenario/

This link without any additional parameters takes us to a single team scenario, location and country are selected randomly.



http://localhost:8000/scenario/?location=r

To set location, select from:

  • h: Home (private home, not compound)
  • oc: Office or organization compound/residence
  • ps: Project site (village, camp, distribution point, hospital, etc.)
  • p: Other public location (street, market, restaurant, etc.)
  • r: Road (in transit)
  • c: Custody (official forces/police)
  • *: random (default if not supppied)


http://localhost:8000/scenario/?country=iraq

To set country, select from:

  • iraq: Iraq
  • syria: Syria/Syrian Arab Republic
  • yemen: Yemen
  • *: random (default if not supplied)


Both country and location can be set togther: http://localhost:8000/scenario/?country=syria&location=r



For management view (multiple teams view)

http://localhost:8000/scenario/?country=syria&location=*,r,c,h,ps,*,*

http://localhost:8000/scenario/?country=syria,iraq,yemen&location=*,r,c,h,ps,*,*



location vector is the base that decides the number of teams. Corresponding countries are taken from the country vector if supplied, otherwise countries are selected randomly.



In all cases we can set the day and/or month, otherwise their default values are of today’s.

http://localhost:8000/scenario/?country=syria,iraq,yemen&location=*,r,c,h,ps,*,*&day=5&month=1



To be able to generate the same missions (taking into consideration that all inputs will remain the same), a seed value could be supplied as follows:

http://localhost:8000/scenario/?country=syria,iraq,yemen&location=*,r,c,h,ps,*,*&day=5&month=1&seed_value=1

Seed value takes any number.

The seed value appears on the top of the page all the time, to be used later for regenration .



To conclude, all possible parameters are:

  • seed_value
  • location
  • country
  • day
  • month


API:

API could be accessed via:

http://localhost:8000/scenario_api/

All possible parameters used with web view could be fed to the API (Only for single scenario) via GET:

  • seed_value
  • location
  • country
  • day
  • month

In addition the API could take the following additional parameters:

  • last_mission_level
  • mean_score
